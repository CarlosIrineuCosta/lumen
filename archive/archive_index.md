## Archival Metadata

### Archival Criteria
- **Date-based**: Files with specific dates that are no longer current
- **Superseded**: Content replaced by newer documentation
- **Legacy**: Approaches or systems no longer in use
- **Temporary**: Editor-specific or working files not meant for long-term storage

### Preservation Notes
- All content is preserved, not deleted
- Git history maintained for additional context
- Archive structure provides organized access
- Index enables search and discovery

### Access Guidelines
- Use this index to locate historical information
- Refer to archive when needing historical context
- Current documentation in `/docs/core/` reflects current state
- For questions about archived content, check git history

## Maintenance

### Regular Reviews
- Monthly: Review new files for archival candidates
- Quarterly: Update archive structure and index
- Annually: Review entire archive for relevance

### Adding to Archive
1. Move file to appropriate archive subdirectory
2. Update this index with new entry
3. Update any references in current documentation
4. Commit changes with "Archive: [filename]" message

### Restoring from Archive
1. Locate file in this index
2. Check git history for additional context
3. Copy file back to active documentation if needed
4. Update documentation references accordingly
